How This Works
Facebook, Instagram, and LinkedIn posting for 12SGI's own channels runs through a self-hosted, free Postiz instance on this machine (http://localhost:4008). This page does not connect anything by itself: it links you straight to each step. Nothing posts anywhere until you finish the OAuth connection for a channel inside Postiz and copy its integration id into config/own_channels.json.
Postiz is bound to 127.0.0.1 only (security choice, see docker-compose.postiz.yml). The "Open Postiz" links below only work when opened on this machine, not from your phone or another Tailscale device. If you want to connect channels from another device, say so and we can talk through exposing it over Tailscale deliberately.
Facebook & Instagram
📘Meta (Facebook Page + Instagram Business)not connected
  1. Create a free app at developers.facebook.com/apps (Business type). No cost — this is your own app, just for API access.
  2. Add the Facebook Login and Instagram Graph API products to that app.
  3. Copy the App ID + App Secret, then set them as FACEBOOK_APP_ID / FACEBOOK_APP_SECRET env vars and restart the Postiz stack (command below).
  4. Open Postiz → Settings → Integrations → Add Channel → Facebook/Instagram → sign in with your 12SGI Page's Facebook account.
  5. Copy the new channel's integration id into config/own_channels.json, set "enabled": true.
Open Postiz → Meta for Developers →
LinkedIn
💼LinkedIn Company Pagenot connected
  1. Create a free app at the LinkedIn Developer Portal, associated with your 12SGI Company Page.
  2. Request the Share on LinkedIn / Community Management API product (free, LinkedIn reviews the request).
  3. Copy the Client ID + Client Secret into LINKEDIN_CLIENT_ID / LINKEDIN_CLIENT_SECRET env vars, restart the Postiz stack.
  4. Open Postiz → Settings → Integrations → Add Channel → LinkedIn → sign in and select the Company Page.
  5. Copy the integration id into config/own_channels.json, set "enabled": true.
Open Postiz → LinkedIn Developer Portal →
After Connecting
Restart Postiz after setting new env vars, then check the queue on your device with the CLI below. Once a channel's config/own_channels.json entry is real (not a PASTE_-prefixed placeholder), the same approve → publish pipeline that's already staging drafts today will start actually posting to it — no code changes needed.
restart: docker compose -f docker-compose.postiz.yml up -d
channel status: python watchers/own_channel_post.py --status
publish a specific approved draft: python tools/publish_approved_social.py --job-id <id> --batch config/social_drafts/<file>.json --platform facebook
