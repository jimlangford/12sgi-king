# County Service Module — Schema & Folder Convention

**Owner:** local_b2a380ef (digital-twin architecture) · **Posted:** 2026-06-18 · **Engine:** `tools/service_module_engine.py`
**For:** the ingestion session (cowork-title19-ingest) + King-server (publish). This answers the coordination request in the dispatch log: *fields + folder convention*. Align per-title structuring to this; no fork needed — your existing `Title / SOURCE URL / RETRIEVED / TYPE` backups map straight in.

Adding a new MCC title or county service = **author a config + drop sourced content → run the engine**. No bespoke HTML.

## Folder convention

```
king_public_src/civic/templates/<id>-service/
  ├─ <id>.config.json        # the module declaration (this schema)
  ├─ <Title NN> Service.html  # GENERATED by the engine — never hand-edit
  ├─ corpus/                  # sourced content (your harvest lands here)
  │   ├─ raw/                 # verbatim snapshots (your backups)
  │   ├─ structured/          # tables/enforcement as JSON/MD
  │   └─ ai_assistant/        # permit-assistant seed (Tier-3)
  └─ analysis/                # LABELED analysis (rules→code, who-pays)
```
`<id>` = `title16`, `title05`, `title03-rpt`, etc. The engine writes the page; `build_site.py` copies `king_public_src/` → `site/king/` and the page publishes at
`/12sgi-king/king/civic/templates/<id>-service/<Title NN>%20Service.html`.

Run: `python tools/service_module_engine.py <id>.config.json` (add `--check` to validate without writing).

## Three layers (each maps to a guardrail)

| Layer | Config carries | Guardrail |
|---|---|---|
| **A. Content** (sourced) | `sections[]` of type `process` / `table` / `parcel-lookup` — each with a `source` | no fabricated law |
| **B. Analysis** (labeled) | `charter-lens` / `analysis-list` / `model` — auto-rendered with an "analysis, not law" label | charter/who-pays labeled |
| **C. Transaction** (gated) | `parcelLookup.mappsPermitUrl`, `tier-gate` section | MAPPS handoff; Stripe env-ref; verified-status only |

```jsonc
  "hero":   { "lead": "...HTML allowed..." },

  // LAYER C — shared MAPPS + GIS lookup component (omit to skip the lookup)
  "parcelLookup": {

    "codeUrl": "https://library.municode.com/.../TIT16BUCO",

  "sections": [ /* ordered; each is one of the typed blocks below */ ],
  "footer": "...HTML..."

```

### Section block types

| `type` | Layer | Key fields |
|---|---|---|
| `parcel-lookup` | A/C | `title`, `note`, `source` |
| `process` | A | `title`, `note`, `steps[{name,what,pay}]`, `charterLens?`, `source` |
| `table` | A | `title`, `columns[]`, `rows[][ "v"|{v,cls,html} ]`, `source`, `badge?` |
| `cards` | A | `title`, `cards[{title,body,href,lahaina?,flag?}]`, `source` |
| `prose` | A | `title`, `html`, `note?`, `source`, `badge?` |
| `charter-lens` | B | `title`, `mauiCharter`, `sovereignCharter`, `position{label,text}` — always labeled analysis |
| `analysis-list` | B | `title`, `intro`, `items[]`, `source` — renders the ⚖️ "not law" callout |
| `model` | B | `title`, `label`, `text` — "proposed model / dated opinion" |
| `tier-gate` | C | `text` — Tier-1-free / Tier-3-verified box |
| `raw` | — | `html` escape hatch |

**`source` object:** `{ "text": "...", "html": "...with <a>...", "expanding": "what's still being filled" }`.
This is exactly where your harvest header maps:

```
Title       -> the section it feeds (process/table/...)
SOURCE URL  -> source.html  <a href="URL">…</a>
RETRIEVED   -> include in source.html text ("retrieved 2026-06-17")
TYPE        -> law => section type process/table (Layer A, must have source)
               opinion => type "model" (dated-opinion label)
               analysis => type charter-lens / analysis-list (Layer B)
```

## Guardrails enforced by the engine

- **No Stripe secret keys** — `guardrail_check()` fails the build if `sk_live_/sk_test_` appears. Use env refs (e.g. `env:STRIPE_PRICE_ASSIST`) in `tier-gate` text only.
- **Layer A needs a source** — `process`/`table` sections that assert code content warn if `source` is missing.
- **Verification model** — Tier-3 text states identity is verified via the processor; only **verified-status** is stored, never raw ID.
- **Theme baked in** — Yale-blue palette, serif body, iPhone-15 (`@media 430px`) + iPad (max-width 980). Don't restyle per module.
- **MAPPS, not replacement** — the page hands off to MAPPS for the official record; it never claims to be the system of record.
- **CPU/web only** — engine is stdlib-only; never touches GPU/ASHES/mesh; writes no secrets.
