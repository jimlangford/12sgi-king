## Token Source of Truth

All design tokens are defined as CSS custom properties in `element_lotus_public/studio.css`.
That file is the **single source of truth** for the Element Lotus visual language.

### Current tokens

```css
/* Backgrounds */
--bg:       #161209   /* page background */
--panel:    #231d10   /* card / panel fill */
--panel-2:  #2e2716   /* metric / nested panel */
--panel-3:  #3e351f   /* tertiary panel */
--line:     #5a4f33   /* borders / dividers */

/* Brand colours */
--gold:     #e3ad33   /* primary brand accent */
--gold-hi:  #f4c95d   /* highlighted / heading gold */

/* Type */
--ink:      #f7f1e3   /* primary text */
--ink-2:    #d8cdb0   /* secondary text */
--ink-3:    #ab9e7f   /* tertiary / label text */

/* Quadrant colours (Pulse Geometry) */
--mauka:    #4ade80   /* upland / growth */
--kula:     #fbbf24   /* field / community */
--makai:    #38bdf8   /* ocean / restoration */

/* Typography */
--sans:     system-ui, -apple-system, "Segoe UI", sans-serif
--serif:    Georgia, "Iowan Old Style", serif

/* Shape */
--r:        12px       /* standard border-radius */
```

## Token Governance

### Adding a new token

1. Define it in `element_lotus_public/studio.css` under the relevant category comment.
2. Document it in the table above with its semantic intent.
3. If the token is needed on a civic surface, copy the variable definition into the
   relevant civic CSS file — do **not** import `studio.css` directly from civic pages,
   as those pages have independent build paths.
4. Open a PR; reviewer confirms the token is not a one-off colour but a reusable
   semantic intent.

### Promoting a pattern from Element Lotus to civic surfaces

Element Lotus is the innovation surface. A pattern is promoted when:

- It has been in production on at least one Element Lotus page for ≥ 2 weeks.
- It meets WCAG 2.2 AA contrast requirements (use a contrast-ratio checker).
- The pattern is documented here with an example reference.

Do not backport civic changes to Element Lotus — the direction is always
Element Lotus → civic, never the reverse.

## Accessibility Standard

All UI surfaces in this system target **WCAG 2.2 Level AA**.

Key requirements:
- Text contrast ratio ≥ 4.5:1 (body text); ≥ 3:1 (large text / UI components).
- Focus indicators must be clearly visible on all interactive elements.
- Touch targets ≥ 44×44 px.
- Avoid using colour alone to convey meaning.

The current token palette passes contrast checks at the defined combinations:
- `--ink` on `--bg` → passes AA (ratio > 7:1).
- `--gold` on `--bg` → passes AA large text (ratio ≈ 5.2:1); fails small body text — do not use `--gold` as body text colour.
- `--gold-hi` on `--bg` → passes AA (ratio ≈ 6.5:1).
- `--ink-2` on `--panel` → passes AA (ratio ≈ 4.7:1).
- `--ink-3` on `--panel` → marginal — use only for non-essential labels (kicker, eyebrow); never for body copy.

## WordPress Layer

The WordPress public layer receives a pre-built Element Lotus bundle generated by
`watchers/deploy_elementlotus_wp.py`. Tokens in the bundle are embedded inline; WordPress
themes do not override them. Re-run `deploy_elementlotus_wp.py` after any `studio.css` change
and re-apply the bundle in the WordPress admin.
